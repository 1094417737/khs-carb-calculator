================================================================
  一、产品定位
================================================================

面向越野跑/超级马拉松/耐力运动爱好者的「赛事级战术规划工具」。
上传 GPX/FIT/KML/KMZ 轨迹文件后，系统自动完成：

  - 坡度感知配速推算（GAP / Naismith's Rule 类算法）
  - 智能补给航点寻路（35min 冷却期 + 评分机制 + 补给匹配）
  - 地图可视化（坡度热力渐变轨道 + 等高线地形 + 方向箭头）
  - 交互式海拔剖面图（触摸滑动查看逐点数据）
  - 分段配速策略建议 + 坡度分布统计
  - 导出带语义化航点的手表兼容文件（GPX / TCX）
  - PWA 离线支持 + 地图瓦片预缓存（深山断网可用）

纯前端 MVP，所有计算在浏览器完成，零网络请求。
支持独立安装到手机桌面（PWA standalone）。

================================================================
  三、完整功能清单
================================================================

【3.1 文件解析】
  - 支持格式：GPX / FIT / KML / KMZ
  - 拖拽上传，最大 10MB
  - DOMParser 前端解析，逐点提取 lat/lon/ele/time
  - 全文件高程缺失时自动填充上游值
  - 解析后原始 GPX 文本完整保留（用于无损导出）

【3.2 核心引擎】
  - 坡度计算引擎 (gradient.ts)
    · 逐点计算相邻轨迹点的高差与水平距离
    · Ramer-Douglas-Peucker 轨迹简化算法（压缩至约 500 点）
    · 五级坡度分类：陡下/缓下/平路/缓上/陡上

  - GAP 时间推演引擎 (gap.ts)
    · 等效平地距离计算：每爬升 100m = +1km
    · 坡度配速系数：上坡 0-15% 线性减速至 55%
    · 陡下坡（<-15%）刹车惩罚至 70%
    · 缓下坡适度提速
    · 强制对齐用户目标完赛时间（timeElapsed 等比缩放）

  - 智能补给航点寻路 (waypoint.ts)
    · 35 分钟最小冷却期
    · ±10 分钟搜索窗口内选评分最高点
    · 评分机制：平路 +50、上坡前 +30、长下坡前 +20、陡坡 -100
    · 补给品匹配：优先胶填补热量+碳水，盐丸填补钠缺口
    · 单点吸收上限：≤2 胶 + ≤3 盐丸 + ≤45g 碳水
    · CP 点感知：接近官方 CP 站时自动清零累积缺口 + 重置冷却期
    · 空匹配时冷却期不空转，缺口继续累积至下一点重试

  - 生理消耗模型 (nutrition.ts)
    · 热量：基于 ACSM MET 模型（MET=7.5, kcal/h = 体重×7.5）
    · 碳水：基线 0.6g/kg/h, 钳制 30-90g/h（基于 Jeukendrup 2014）
    · 钠：基于出汗率估算 + 汗液钠浓度（Sawka 2007）

【3.3 地图可视化】
  - CartoDB Voyager 暗色/亮色双主题底图
  - OpenTopoMap 等高线地形叠加层（opacity 0.35）
  - 多段渐变色轨迹线（紫红=上坡、青蓝=下坡、灰白=平路）
  - 起终点标记（绿 S / 红 E）+ 方向箭头
  - 补给航点标记（⚡能量胶/💧盐丸/⚡💧组合/🍫固体）
  - 自定义打点（🚩普通标记/🏁CP 点标记）
  - 点击地图吸附至最近轨迹点（100m Haversine 容差）
  - 自定义打点弹窗（替代 window.prompt，含 CP 点开关）

================================================================
  五、关键算法详解
================================================================

【5.1 GAP 坡度配速系数 (gap.ts)】
  基准：平路配速 = 目标完赛时间 / 等效平地距离
  等效平地距离 = 实际距离 + 爬升/100 - 缓下坡/100×0.15 + 陡下坡×0.3/100

  速度系数 speedMultiplier(gradient):
    上坡 0-15%: 1.0 → 0.55 (线性递减)
    上坡 >15%: 0.55 (步行)
    平路 -3~0%: 1.0
    缓下坡 -15~-3%: 1.0 → 1.2 (加速)
    陡下坡 <-15%: 0.70 (刹车)

【5.2 补给航点寻路 (waypoint.ts)】
  输入：带 timeElapsed 的轨迹点 + 代谢率(kcal/min, 钠/min, 碳水/min)
  输出：Waypoint[] 按距离排序

  算法步骤：
  1. 遍历轨迹点，逐分钟累积热量/钠/碳水缺口
  2. 冷却期 >35min + 缺口 >半份胶热量 → 触发搜索
  3. ±10min 窗口内找评分最高的点（评分函数见下）
  4. 匹配补给品（胶优先→盐丸），受单点上限约束
  5. 减去已覆盖缺口，剩余缺口顺延至下一区块
  6. 遇到 CP 点：强制清零缺口 + 重置冷却期

  评分函数 scorePoint():
    平路(-3%~3%) +50
    缓坡(-8%~8%) +25
    陡坡(>15%或<-15%) -100
    前方上坡 >10% +30
    前方下坡 <-10% +20

【5.3 生理消耗模型 (nutrition.ts)】
  kcal/h = 体重(kg) × 7.5 (MET)
  carbsG/h = clamp(体重×0.6, 30, 90)
  sodiumMg/h = 0.5L/h出汗 × 500mg/L × 0.7 (补70%流失)
               + 温度修正(>20°C加增量) + 体重修正(>70kg加增量)

【5.4 GPX 无损注入导出 (export-gpx.ts)】
  策略：不上传原始 GPX → 回退重建
       上传原始 GPX → 保存 rawGpxText → 在 </gpx> 前注入 <wpt> 节点
  红线：绝不修改 rawGpxText 中的 <trkpt> 及其子节点

【5.5 轨迹简化 (gradient.ts)】
  算法：Ramer-Douglas-Peucker
  输入：TrackPoint[]（可含 10000+ 点）
  输出：≈500 点
  保留特征：拐弯处和大坡度变化处优先保留

================================================================
  六、状态管理架构
================================================================

TrailContext (useReducer) 全局状态：

  TrailState:
    gpxFileName: string           # 文件名
    trackPoints: TrackPoint[]     # 轨迹点数组
    rawGpxText: string | null     # 原始 GPX XML（无损导出用）
    userProfile: {                # 用户画像
      targetTimeMinutes: number
      tempC: number
      weightKg: number
    }
    nutritionLibrary: []          # 补给库
    waypointConfig: {}            # 补给策略配置
    result: TrailResult | null    # 计算结果
    step: TrailStep               # 当前步骤 (upload|configure|results)
    safeMode: boolean             # 英文安全名开关
    customMarkers: []             # 自定义标记（含 isCheckpoint）
    distanceOverrideKm: number|null
    climbOverrideM: number|null
    pendingMarkerInput: {}|null   # 打点浮层状态

  持久化至 localStorage (trail_config_v6):
    userProfile, waypointConfig, nutritionLibrary, safeMode

  useEffect 自动重算：
    trackPoints 变化 → 调用 computeTrail() → dispatch SET_RESULT
    依赖项：trackPoints, userProfile, waypointConfig,
            nutritionLibrary, distanceOverrideKm, climbOverrideM,
            customMarkers

  十、已知局限与未来方向
================================================================

  当前局限：
  - 仅支持 GPX/TCX 导出，未支持 FIT 格式导出
  - 地图瓦片依赖 CartoDB/OpenTopoMap 免费服务，国内访问较慢
  - 无 Strava/TrainingPeaks 一键分享
  - 无用户账号/云端同步

  可扩展方向：
  - FIT 格式导出（Garmin 原生格式）
  - 预设补给方案（50K/100K/炎热天气一键加载）
  - 心率/功率数据叠加显示
  - Strava API 集成
  - 多人协作/分享路线
