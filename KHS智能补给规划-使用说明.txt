============================================================
  一、产品概述
============================================================

KHS 智能补给规划是一款面向越野跑/耐力运动爱好者的纯前端工具。
上传 GPX 轨迹文件后，系统根据地形坡度自动推算分段配速，
在平缓路段智能放置补给航点，并匹配能量胶/盐丸补给方案。
最终导出带航点的手表兼容文件（GPX/TCX），可直接导入
Garmin / COROS / Suunto 等运动手表导航使用。

完全离线可用（PWA），所有计算在浏览器完成，零网络请求。

============================================================
  二、两种模式
============================================================

顶部 Header 有两个 Tab 可切换：

【补碳计算】— 原有功能
  基于运动科学的个性化耐力补给方案计算器。
  输入体重、运动时长、强度等参数，计算碳水化合物补充量。

【Trail补给】— 越野跑智能规划（本说明书重点介绍）
  上传轨迹 → 自动推演配速 → 智能放置补给点 → 导出导航文件。

============================================================
  三、Trail补给 — 使用流程
============================================================

Step 1：上传轨迹文件
  - 支持格式：GPX / FIT / KML / KMZ
  - 拖拽文件到上传区域，或点击选择文件
  - 最大支持 10MB
  - 前端 DOMParser 逐点解析 lat/lon/ele

Step 2：设置运动参数
  - 体重（kg）：用于计算能量消耗
  - 目标完赛时间（HH:MM）或平均配速（min/km）
  - 距离和累计爬升自动从轨迹计算，可点击数字手动校准

Step 3：查看结果与导出
  - 地图展示渐变轨迹线（紫红=上坡、青蓝=下坡、灰白=平路）
  - 补给航点自动生成并显示在地图上
  - 海拔剖面图展示全程地势变化
  - 导出 GPX / TCX 文件导入手表

============================================================
  四、核心功能详解
============================================================

【4.1 数据面板】（运动参数卡片内）
  - 轨迹距离 / 等效平地距离
  - 累计爬升 / 累计下降
  - 预估完赛耗时 / 平均配速
  - 代谢消耗：kcal/h、碳水 g/h、钠 mg/h
  - 距离和爬升可点击直接编辑（用于校准 GPS 偏差）

【4.2 渐变轨迹地图】
  - 暗色/亮色双主题自适应
  - 多段渐变色轨迹线，颜色随坡度变化：
    紫红 ＝ 上坡（陡）
    青蓝 ＝ 下坡（陡）
    灰白 ＝ 平路/缓坡
  - 起终点标记（绿色 S / 红色 E）
  - 方向箭头指示行进方向
  - 补给航点标记（⚡能量胶 / 💧盐丸 / ⚡💧组合）

【4.3 补给库管理】
  - 自定义添加补给品：能量胶 / 盐丸 / 固体
  - 设置每件的 kcal、碳水克数、钠含量
  - 同类型单选激活：只有打勾（●）的物品参与计算
  - 同一类型只能激活一项（互斥逻辑）

【4.4 智能补给航点算法】
  核心引擎自动完成以下步骤：

  a) 坡度计算 — 逐点计算轨迹坡度
  b) 轨迹简化 — Ramer-Douglas-Peucker 算法压缩至约 500 点
  c) GAP 推演 — 基于坡度调整配速（上坡减慢、下坡加快）
  d) 航点寻路 — 35 分钟冷却期 + ±10 分钟搜索窗口：
     - 评分机制：平路 +50、上坡前 +30、长下坡前 +20、陡坡 -100
     - 在搜索窗口内选评分最高的位置放置补给点
  e) 补给匹配 — 基于热量/碳水/钠缺口匹配补给品
     - 能量胶优先填补热量+碳水缺口
     - 盐丸填补钠缺口
     - 单点吸收上限约束

【4.5 自定义打点】
  - 点击地图任意位置，自动吸附至最近轨迹点（100 米容差）
  - 弹出浮层输入标记名称（如：危险下坡、取水点、CP3）
  - 支持中文/英文/数字混合命名
  - 自定义标记以 🚩 图标显示在地图上

【4.6 导出功能】
  - GPX 导出：含 <wpt> 航点，兼容 Garmin/COROS/Suunto
  - TCX 导出：含 <CoursePoint> 航点
  - 「英文安全航点名」开关：
    内置 14 组关键词智能映射字典，自动将中文标记转为英文：
    CP/打卡→CP、补给站→AS、起点→START、终点→FINISH、
    水/泉→WATER、食/饭→FOOD、医/救→MEDIC、险/崖→DANGER、
    顶/峰→PEAK、桥→BRIDGE、路口/岔→TURN、风景→VIEW、
    杖→POLES、包/换装→BAG、厕→WC
  - 自定义标记名中的阿拉伯数字自动保留（如 CP3→CP-3）
  - 纯英文名自动转大写
  - 中文名无匹配时回退为 WPT-1, WPT-2…
  - 航点计数器：>50 黄色警告、>100 红色警告
  - XML 控制字符自动过滤，确保文件安全

【4.7 海拔剖面图】
  - Canvas 渲染，支持暗色/亮色主题
  - 显示全程海拔变化曲线
  - 补给航点在剖面上标注

============================================================
  六、常见问题
============================================================

Q: 地图加载太慢怎么办？
A: 地图瓦片依赖 CartoDB 服务，国内访问可能较慢。
   不影响核心功能——轨迹、航点、图表均为纯前端渲染，
   即使地图未完全加载，补给方案计算也已完成。

Q: 能不能用高德/腾讯地图？
A: 越野跑轨迹分布在全球各地，国内地图商 95% 以上
   是山区野外无数据。Leaflet 的 CartoDB 瓦片覆盖全球
   且免费、无需 API Key。

Q: 导出的 GPX 文件如何导入手表？
A: 以 Garmin 为例：
   Garmin Connect 网页版 → 训练 → 路线 → 导入
   或通过 Garmin BaseCamp 导入后同步。

Q: 补给航点为什么不在我期望的位置？
A: 算法基于 35 分钟冷却期自动规划，调整完赛时间或
   补给品参数可改变航点分布。如需精确指定位置，
   使用地图点击打点功能手动添加。
